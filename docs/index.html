The interesting part

It gets better at getting better.

Every interaction leaves a signal. Four times a day, the evolution engine reviews them and decides what to improve next.

01

Sense

Every interaction is scanned for signals — missed searches, manual fallbacks, user corrections. No LLM needed.

02

Gather

Accumulated signals are clustered into evolution candidates: fixes, extensions, improvements, integrations.

03

Rank

Each candidate is scored by impact and feasibility. The top candidates survive. Max 2 executions per cycle.

04

Execute

The engine reads relevant source code, generates a patch, runs tests in a sandbox, and opens a PR for review.

05

Verify

After merge, the system monitors whether the original signal recurs. If it does, the candidate re-enters the pipeline.

What counts as a signal?

Search Miss

"I don't have information on that"

Knowledge gap detected → extend action or improve retrieval

Manual Fallback

"Try running this command yourself"

Capability gap → build new action module

User Correction

"No, actually I meant..."

Intent parsing failure → improve context handling

Recurring Failure

Same error 3+ times

Systematic bug → self-healing generates patch

Safety model

Trust, but classify.

Every action is classified before execution. The autonomy controller gates behavior based on risk level.

Read

Always allowed. No approval needed.

search_knowledge get_context search_email cursor_status terminal_status

Write

Requires approval in supervised mode.

send_email create_reminder modify_file terminal_exec cursor_open_project

Dangerous

Always requires explicit approval.

send_money delete_data share_externally generate_capability
